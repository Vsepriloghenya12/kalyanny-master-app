# Кальянный мастер

Мобильное hookah-приложение с отдельной страницей владельца.

## Что уже есть
- мобильная витрина в стиле референса
- автопрокрутка баннеров
- популярные миксы с модальным окном
- каталог продукции, бренды, новинки, подборки
- отдельная страница владельца `/owner`
- управление контентом через API и owner-панель
- стили вынесены отдельно

## Локальный запуск
```bash
npm install
npm run build
npm start
```

После запуска:
- клиент: `http://localhost:8080`
- owner: `http://localhost:8080/owner`

## Railway
Build Command:
```bash
npm install && npm run build
```

Start Command:
```bash
npm start
```

## ENV
Смотри `.env.example`.

## Важно про хранение данных
По умолчанию данные сохраняются в `server/runtime-data/content.json`.
Для Railway это рабочая демо-версия, но файловая система может сброситься после redeploy/restart.
Для постоянного хранения следующим шагом лучше перевести `storage.js` на Postgres.
